# Research & Plan Review — Aggregate Summary

**Scope:** five parallel research/plan artifacts on this repo — `grammata-audit.md`, `research.md`, `analytics_gpt_research.md`, `PRD-team-visibility.md`, and `analytics-research/` (the presearch + PRD + CLAUDE.md + challenger critique pack that seeded the current consolidated PRD).

**Best plan (one-liner):** **The `analytics-research/PRD.md` pack (now consolidated as Bematist) is the strongest base** — it is the only plan that treats grammata as a *building block, not a destination*, commits to a 4-week parallel-workstream execution plan with honest IDE-coverage matrix, adversarial-tested schema, and a deployable OSS + self-host product independent of Pharos. Three layers must be grafted onto it: (a) **accuracy + outcome + waste wedges** from `research.md` (Lane 0 dedup fix is a P0 correctness blocker; outcome attribution, waste radar, per-prompt $ are the uncontested competitive moats); (b) **AI Leverage Score framing + 2×2 manager view + maturity ladder** from `analytics_gpt_research.md` (safer, more defensible than a raw leaderboard); (c) **IC Private Agent Coach + k-anonymity floor + BSL 1.1 open-core split + works-council templates** from `PRD-team-visibility.md` (without its Pharos coupling).

**Common failure across 3 of 5 plans:** `grammata-audit.md`, `research.md`, and especially `PRD-team-visibility.md` treat Pharos as the natural consumer / UI host / IPC target. That coupling is incorrect per the brief — this is an independent greenfield project. Every mention of `pharos link grammata`, "Pharos consumes the same DuckDB", Electron IPC, or "Pharos is the solo-mode dashboard" must be stripped. Grammata is one of several local adapters; Bematist is the product.

**What survives from each plan:**
- `grammata-audit.md` — the complete signal catalogue (54 tracked metrics, 190 untapped session fields). Keep as the adapter-level field reference.
- `research.md` — competitive map (40+ tools), Lane 0 dedup bug evidence, the five buildable wedges, Continue.dev + Copilot IDE + Cline/Roo adapter lanes. Keep as the differentiation playbook.
- `analytics_gpt_research.md` — AI Leverage Score (35/25/20/10/10 weighting), Aware→Operator→Builder→Architect ladder, 2×2 Outcome×Efficiency manager matrix. Keep as the scoring/UX contract.
- `PRD-team-visibility.md` — privacy architecture (counters / envelopes / full), k-anonymity k≥5, on-device DP, IC Coach, BSL 1.1, works-council templates, SOC 2 Type II path, access-control matrix. Keep the privacy+compliance spine; drop the Pharos/Electron skin.
- `analytics-research/` — the full parallel-workstream PoC plan, TruffleHog+gitleaks server-side redaction, decomposed Haiku 4.5 insight pipeline with citation grounding, honest IDE-fidelity matrix, sigstore+SLSA L3 distribution. Keep as the execution plan.

**Net product name, positioning, and license:** **Bematist** (a new independent project), Apache 2.0 for the agent + OSS dashboard, BSL 1.1 → Apache 2.0 after 4 years for the enterprise gateway features (SSO/SCIM/audit-export/DP). Solo-first, team-additive, never tied to any particular desktop wrapper.

## Addendum — `analytics-product/` ("Grammata" Clio-pipeline plan, reviewed 2026-04-16)

**Verdict:** sixth plan reviewed; strong in math, unique in approach, compromised by the same Pharos coupling error. **Not** the winning base plan — it contributes specific upgrades that now fold into `PRD.md`, but its stack (Firebase / Firestore / Cloud Functions) would kill the open-source + self-host + air-gapped positioning locked in by the consolidated PRD, and its UI is described as extensions to `pharos-landing` + `pharos.dev/team/<slug>` + Pharos Electron desktop with specific existing components (`HeroStats`, `CostOverTimeChart`, `WrappedRecap`, etc.) named as the manager view — i.e. the same Pharos-as-the-product mistake as `PRD-team-visibility.md`.

**How it uniquely stacks up:** it is the ONLY plan of the six to fully specify (a) a Clio-adapted on-device **redact → abstract → verify-non-identifying → embed → cluster** pipeline with two LLM passes (Anthropic's Clio paper + OpenClio is the prior art cited); (b) the exact **AI Leverage Score math** (winsorize p5/p95 → cohort percentile rank → weighted 35/25/20/10/10 → confidence `√(outcomes/10)·√(active_days/10)` → `final = raw × confidence`) to an eval-gateable level of precision; (c) an opt-in **`AI-Assisted: grammata-<sessionId>` commit trailer** that solves session→PR attribution without relying on Copilot Metrics API; and (d) **"Promote to team playbook"** as the explicit positive-consent loop that turns privacy-preserving clustering into cross-team sharing signal for the Team Impact subscore. It also contributes the **500-case synthetic dev-month eval** as a scoring-math CI gate (MAE ≤ 3) and the **developer-notified-of-manager-view** transparency primitive (audit-log row per manager drill, daily digest to the IC).

**Pulled into `PRD.md`:**
1. **Clio pipeline** (redact with Presidio + TruffleHog + Gitleaks → abstract via user's own running agent through MCP, Ollama fallback, never cloud LLM on raw prompts → verifier LLM pass "is this identifying? YES/NO", drop on YES → `all-MiniLM-L6-v2` embedder → HDBSCAN cluster with k ≥ 3 contributor floor → cloud Haiku labels only already-redacted abstracts) — upgraded §8 Insight Layer.
2. **AI Leverage Score math** locked as eval-gateable formulas — §7.1 locked constants.
3. **Minimum sample gates** (≥ 10 sessions AND ≥ 5 active days AND ≥ 3 outcome events AND cohort ≥ 8 peers) — §6.4, §14.
4. **`AI-Assisted:` commit trailer** as an opt-in git-hook line that `grammata-github` parses to join session → PR — §8.5.
5. **Developer notified of manager views** as a Bill of Rights line item, audit-log row per drill, daily digest default on — §6.5.
6. **Promote-to-team-playbook** as the Team Impact subscore signal source — §7.1 + §8.2.
7. **500-case scoring eval** CI gate — §14.

**Rejected from `analytics-product/`:**
- Firestore as the event store (proprietary managed service; kills OSS + on-prem + air-gapped).
- Firebase Auth as identity (kills self-host; we keep Better Auth → WorkOS).
- Cloud Functions + Cloud Scheduler as compute (kills self-host; we keep Bun workers + Redis streams + PgBoss crons).
- Pharos-coupled UI at `pharos.dev/team/<slug>` (we own `bematist.dev/team/<slug>` and we own our own components — no reuse of Pharos's 10 existing analytics components).
- "Product lives inside the `pella-labs/pharos` monorepo" (Bematist is an independent repo per the brief).

**Net effect:** the consolidated `PRD.md` now has the first plan's 4-week execution rigor, the second plan's correctness + outcome + waste wedges, the third plan's defensible scoring framing, the fourth plan's privacy architecture, and the sixth plan's Clio pipeline + locked math + commit trailer + eval gate — with none of the five plans' Pharos coupling or vendor-managed-service traps.
